Project Overview

The Challenge

Early detection of corn diseases (Blight, Common Rust, Gray Leaf Spot) is crucial for food security. Traditional manual inspection is slow and error-prone. We aimed to build an automated, efficient diagnostic tool accessible via web browser.

The Solution & My Contribution

We leveraged the state-of-the-art YOLOv8s-cls architecture. I led the experimental phase, training three model variants: Original (224px), Enhanced Augmentation, and Modified Input (320px).

Surprisingly, the Original configuration outperformed others with an accuracy of 96.7% and the fastest inference time (~12.7ms), proving that bigger inputs don't always mean better results for specific datasets. I then deployed this optimized model into a user-friendly Streamlit interface.

Key Takeaways

This project taught me the art of model selection and trade-offs. While higher resolution inputs usually improve detection, they drastically increased inference time (+86%) with negligible accuracy gain in this case. I learned to prioritize efficiency for deployment over raw theoretical complexity.
